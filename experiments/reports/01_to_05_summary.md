# Summary of Experiments 01–05

This analysis evaluated the warehouse object detection pipeline across model selection, dataset sampling, NMS threshold design, augmentation robustness, and hard-negative-mining strategy. Across the five analyses, the main conclusion is that Model 2 is the stronger baseline detector, but deployment-quality performance depends on careful post-processing, representative sampling, robustness testing, and targeted failure mining.

Model 2 should be used as the baseline because it produced higher mAP@0.5 than Model 1 and improved AP across all object classes. However, the class-level results also show that aggregate mAP is not sufficient by itself. Some safety-relevant and operationally important classes remain weaker in absolute terms, so future work should continue to monitor per-class behavior rather than relying only on the overall mAP score.

The rare-aware density-stratified 5,000-image sample is a defensible working subset because it preserves the full dataset’s class distribution, rare-class coverage, object-density structure, and crowding patterns. This matters because later design decisions, especially NMS thresholding and hard-negative mining, depend heavily on whether the sample includes sparse scenes, dense scenes, rare objects, and overlapping objects.

For post-processing, the recommended NMS IoU threshold is 0.50. Although a threshold of 0.60 produced a slightly higher aggregate mAP, the gain was too small to justify the increase in duplicate-like detections. Threshold 0.50 provides the best engineering trade-off because it keeps performance near the maximum while preserving a cleaner detection output.

The augmentation analysis showed that Model 2 is relatively stable under brightness changes but vulnerable to Gaussian blur and especially vertical flip. This suggests that future robustness work should prioritize realistic warehouse distortions such as blur, lighting variation, focus issues, motion artifacts, and compression effects. Vertical flip should be used cautiously because it caused the largest degradation and may not represent a realistic camera condition unless inverted camera views are expected.

The hard-negative-mining analysis showed that lambda values can control which failure modes are emphasized, but only if loss-component scale is handled carefully. Raw HNM scoring was dominated by localization loss, so all main lambda settings selected the same images. After scale-aware normalization, the lambda settings behaved as intended: localization-heavy sampling selected poor box-placement cases, objectness-heavy sampling selected weak matched detections, no-object-heavy sampling selected false-positive-heavy images, and classification-heavy sampling selected class-confusion cases. The main HNM limitation is that the current prediction-based loss does not directly mine zero-prediction false negatives, so it should be paired with a separate missed-detection mining strategy.

Overall, the strongest Warehouse Object Detection configuration from this analysis is Model 2 with the rare-aware density-stratified sample for experimentation, NMS IoU threshold 0.50 for post-processing, brightness and blur-aware robustness planning, and scale-aware HNM for targeted rectification. The system is promising, but it should not be treated as deployment-ready based only on aggregate mAP. A reliable warehouse detection system should continue to use per-class evaluation, duplicate-detection monitoring, robustness testing, and failure-mode-specific data mining before deployment.
